1. generate_lead

Página: /contacto

Justificación Estratégica:
El formulario de contacto es el principal punto de conversión del sitio web, transformando a un visitante anónimo en un lead conocido. Medir cada envío exitoso es fundamental para evaluar la eficacia de las campañas de marketing, calcular el coste por lead y entender qué canales generan más oportunidades de negocio.
Estrategia de Captura:
El formulario se carga dinámicamente vía JavaScript y su envío se procesa con AJAX. El trigger nativo de GTM 'Envío de formulario' no funcionará. La estrategia robusta es usar un trigger de 'Visibilidad de elemento' que detecte la aparición del mensaje de éxito tras el envío. Este método es fiable para formularios AJAX.

2. contact_interaction

Página: /contacto

Justificación Estratégica:
Medir las interacciones de contacto directo es fundamental para entender cómo los usuarios convierten su interés en una acción concreta. Este evento nos permite diferenciar qué canales de contacto (teléfono, email, redes sociales) son más efectivos, optimizar la página de contacto y proporcionar al equipo de marketing datos claros sobre la generación de leads directos, además de alimentar el píxel de Facebook con señales de alta intención.
Estrategia de Captura:
Se utiliza un único activador de 'Solo enlaces' que se dispara en las páginas de contacto. Una variable JavaScript personalizada ('JS - Contact Method') inspecciona la URL del clic para identificar el tipo de interacción (tel:, mailto:, social media). Esto centraliza la lógica y simplifica el activador, haciéndolo más robusto.

Variables JavaScript

JS - Contact Method (JavaScript personalizado) 
function() {
  var url = {{Click URL}};
  if (!url) return undefined;
  
  if (url.indexOf('mailto:') === 0) {
    return 'email';
  }
  if (url.indexOf('tel:') === 0) {
    return 'phone';
  }
  if (url.indexOf('instagram.com') !== -1) {
    return 'instagram';
  }
  if (url.indexOf('linkedin.com') !== -1) {
    return 'linkedin';
  }
  if (url.indexOf('facebook.com') !== -1) {
    return 'facebook';
  }
  
  return undefined;
}

Devuelve: El método de contacto basado en la URL del clic: 'email', 'phone', 'instagram', 'linkedin', 'facebook', o undefined si no coincide.

JS - Click Domain (JavaScript personalizado) 
function() {
  try {
    var url = {{Click URL}};
    if (!url || url.indexOf('mailto:') === 0 || url.indexOf('tel:') === 0) return undefined;
    var start = url.indexOf('://');
    if (start === -1) return undefined;
    var domain = url.substring(start + 3);
    var slash = domain.indexOf('/');
    if (slash !== -1) domain = domain.substring(0, slash);
    if (domain.indexOf('www.') === 0) domain = domain.substring(4);
    return domain;
  } catch(e) { return undefined; }
}

Devuelve: El nombre de dominio limpio de una URL (ej: 'facebook.com'). Devuelve undefined para enlaces mailto: y tel:.

3. navigation_click

Página: /contacto

Justificación Estratégica:
Medir los clics en la navegación principal es fundamental para entender cómo los usuarios interactúan con el sitio. Este evento permite identificar qué secciones son las más populares, qué elementos generan más interés y si existen puntos de fricción en la navegación. Los datos recogidos son clave para optimizar la arquitectura de la información y mejorar la experiencia de usuario.
Estrategia de Captura:
Se utilizará una etiqueta Custom HTML con un listener de eventos de clic en el
. Esta estrategia es necesaria para diferenciar de forma fiable entre los distintos tipos de menús (primario, social, idioma) y para extraer el texto del enlace correctamente, incluso si el usuario hace clic en un elemento hijo (un o dentro del ). El script empujará los datos contextuales al dataLayer, que serán leídos por variables y enviados a GA4.
